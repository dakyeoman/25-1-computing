###  ** 주제 제출 내용 **
1) 문제명: 서울시 창업 입지 선정 최적화를 위한 다중 기준 의사결정 알고리즘

2) 문제 설명: 서울시 열린데이터광장 API를 활용하여 다목적 최적화와 제약조건 만족 문제(CSP)가 결합된 창업 입지 추천 시스템을 구현하라

a) 핵심 알고리즘 요구사항

다목적 최적화: 파레토 최적해 탐색 (수익성 vs 안정성 vs 접근성)

제약조건 만족: 예산, 경쟁수준, 타겟매칭 제약 하 최적해 탐색

b) API 정보

인구 데이터 API: http://openapi.seoul.go.kr:8088/{API_KEY}/json/citydata_ppltn/1/5/{지역명}

상권 데이터 API: http://openapi.seoul.go.kr:8088/{API_KEY}/json/citydata_cmrcl/1/5/{지역명}

지원 지역: 서울시 82개 주요 상권

c) 입력: 창업자 정보areas = ["강남역", "홍대입구역", "명동", "이태원역", "잠실역"]business_type = "카페"target_customers = "직장인 학생"budget_min = 5000000budget_max = 15000000max_competition = 50min_target_match = 60

d) 알고리즘 구현 요구사항1. 다목적 최적화

3개 목적함수 정의:

수익성: 0.4 × 유동인구점수 + 0.4 × 결제활성도점수 + 0.2 × 타겟매칭점수  

안정성: 0.6 × 적정경쟁점수 + 0.4 × 예산적합도점수  

근성: 0.6 × 비거주민비율 + 0.4 × 상권레벨점수

파레토 지배관계: 해 A가 해 B를 지배한다 ⟺ 모든 목적함수에서 A ≥ B이고, 적어도 하나에서 A > B

파레토 최적해 집합: 다른 해에게 지배당하지 않는 해들의 집합

2. 제약조건 만족 문제 (CSP)

제약조건:

예산 제약: budget_min ≤ 지역평균결제금액 ≤ budget_max경쟁 제약: 해당업종매장수 ≤ max_competition타겟 제약: 타겟고객매칭률 ≥ min_target_match

백트래킹 알고리즘: 제약조건을 위반하는 해는 즉시 제거하여 탐색 공간 축소

e) 데이터 처리 및 점수 계산API 응답 데이터 활용

인구 데이터:

AREA_PPLTN_MIN/MAX: 유동인구 규모PPLTN_RATE_20/30: 20-30대 비율 (타겟매칭)NON_RESNT_PPLTN_RATE: 비거주민 비율 (접근성)상권 데이터:AREA_SH_PAYMENT_CNT: 결제건수 (활성도)AREA_SH_PAYMENT_AMT_MIN/MAX: 결제금액 (예산적합도)RSB_MCT_CNT: 업종별 매장수 (경쟁수준)AREA_CMRCL_LVL: 상권레벨 (접근성)

핵심 점수 계산 공식

1. 유동인구 점수: 점수 = min(100, (AREA_PPLTN_MAX / 10000) × 100)

2. 타겟매칭 점수 (직장인+학생 기준)

직장인매칭 = (20대비율 + 30대비율) × 0.8 + 비거주민비율 × 0.2학생매칭 = 10대비율 × 0.2 + 20대비율 × 0.8최종점수 = (직장인매칭 + 학생매칭) / 2

3. 적정경쟁 점수 (역U자형 함수) 적정값 = 40 (카페 기준)현재값 = 해당지역 카페매장수if 현재값 ≤ 적정값:    점수 = 60 + (현재값/적정값) × 40else:    점수 = max(30, 100 - (현재값-적정값)/적정값 × 70)

4. 상권레벨 점수: 매우높음: 100점, 높음: 80점, 보통: 60점, 낮음: 40점, 매우낮음: 20점

알고리즘 순서

API 데이터 수집: 각 지역별 인구/상권 데이터 병렬 수집

제약조건 필터링: CSP 알고리즘으로 제약조건 위반 지역 제거

목적함수 계산: 남은 지역들의 3개 목적함수 값 계산

파레토 분석: 지배관계 확인하여 파레토 최적해 집합 도출

결과 출력: 파레토 프론티어와 추천 순위 제공

출력 예시

=== 제약조건 만족 지역 ===강남역: 예산 적합, 경쟁 적정, 타겟매칭 85%홍대입구역: 예산 적합, 경쟁 적정, 타겟매칭 92%명동: 예산 초과로 제외=== 파레토 최적해 집합 ===1. 강남역: 수익성(92.3), 안정성(78.4), 접근성(85.7)2. 홍대입구역: 수익성(87.6), 안정성(89.2), 접근성(91.4)3. 이태원역: 수익성(83.1), 안정성(91.8), 접근성(94.2)=== 추천 결과 ===종합 1위: 강남역 (높은 수익성 중심)균형 추천: 홍대입구역 (3개 목적함수 균형)안정 추천: 이태원역 (높은 안정성/접근성)
